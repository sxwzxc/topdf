## 大图片 500 错误修复 — 全链路压缩方案

**问题**：上传大分辨率/大文件体积图片时，后端返回 500 (Internal Error)。根源是：
- 前端无压缩，原始文件直传
- ImageEditor 输出未压缩 PNG（裁剪/旋转/矫正后可能很大）
- 后端无尺寸限制，Pillow `DecompressionBombError` 或内存溢出

**修复 (三端加固)**：

### 1. 新增 `src/lib/imageCompressor.ts`
- 最长边 > 4096px → 等比缩放到 4096px
- PNG/WebP → 转 JPEG (quality 85%)
- 已合规的 JPEG → 跳过避免二次损失
- GIF → 保持原样（超大则拒绝）
- 单张 > 50MB → 拒绝并提示
- 导出 `compressImage(file)`, `compressBlob(blob, name)`, `compressImages(files)`, `ImageTooLargeError`, `formatFileSize`

### 2. 修改 `src/app/page.tsx`
- `addFiles` 改为 async，逐张调用 `compressImage`
- 添加 `compressing` / `compressingCount` / `applyingEdit` 状态
- 压缩期间上传区显示 spinner + 进度 "(d/total)"
- 压缩期间禁用拖拽、点击、清空、转换按钮
- `applyEdit` 改为 async，调用 `compressBlob` 压缩编辑器输出
- `handleDrop` / `handleFileChange` 加压缩中守卫

### 3. 修改 `cloud-functions/api/img2pdf.py`
- `MAX_DIMENSION = 4096`：超限自动 LANCZOS 缩放
- `MAX_BODY_SIZE = 50MB`：超限返回 413
- `Image.MAX_IMAGE_PIXELS = 200_000_000`：提高 Pillow 解压阈值
- 捕获 `DecompressionBombError` → 400 + 中文提示
- 捕获 `MemoryError` → 400 + 中文提示
- 捕获 `OSError` → 400 + 中文提示
- `ValueError` → 400，其余 `Exception` → 500

## 四点矫正算法重写 — 仿射 → 透视变换 (Homography)

**原算法**：双三角形仿射近似（`computeAffine` + `warpTriangle` + canvas clip）。
仿射只能做旋转/缩放/剪切，无法表达透视消失点，导致矫正结果严重失真。

**新算法**：Homography 透视变换（DLT 求解 + 反向映射 + 双线性插值）
- `computeHomography(src, dst)` — 4对点 DLT 求解 3×3 透视矩阵
- `solveNullspace(A, n)` — 幂迭代法解 A^T·A 最小特征向量（替代 SVD）
- `invertHomography(H)` — 矩阵求逆用于反向映射
- `warpQuadrilateral` — 逐像素反向映射 + 双线性插值，ImageData 直接操作
- 超出源图范围 → 白色填充

## 色彩修正 / 文档增强功能

ImageEditor 新增「色彩修正」Tab，将手机拍照转为类扫描文档效果。

### 图像处理流水线
1. **白平衡 + 直方图拉伸**：每 RGB 通道独立取 2%~98% 分位，线性拉伸到 0~255（contrast 参数调分位范围）
2. **亮度偏移**：±40% 范围，0.5 = 不调整
3. **Unsharp Mask 锐化**：3×3 高斯模糊核近似，`sharp = orig + (orig - blur) * strength`
4. **灰度转换**：亮度加权 (0.299R + 0.587G + 0.114B)
